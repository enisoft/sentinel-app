# Auditoria ENI-84 — 2026-07-05

## Resumo do pedido

Sessão persiste offline: operador logado continua capturando sem rede mesmo com JWT expirado e refresh Supabase falho. 401 real online ainda desloga.

---

## Relatório por critério

### 1. Permanência offline não abre brecha

| Item | Veredito | Evidência |
|------|--------|-----------|
| Baseia-se em sessão prévia no Keystore | ✅ CONFERE | `SupabaseAuthGateway.hasPersistedSession()` → `supabaseSecureSessionKey` em `SecureKeyValueStore` |
| Logout manual limpa sessão | ✅ CONFERE | `signOut` seta `_manualSignOut = true` e chama `Supabase.auth.signOut()` |
| Não cria sessão nova sem auth | ✅ CONFERE | `canAccessApp` só true com sessão ativa OU Keystore intacto; login exige `signInWithPassword` |
| Não aceita sessão forjada | ✅ CONFERE | Keystore protegido (ENI-33); app offline só reutiliza credencial já persistida pelo SDK |

### 2. AuthGate / appAccessStream

| Item | Veredito | Evidência |
|------|--------|-----------|
| Não cai pro login só por sessionStream null offline | ✅ CONFERE | `AuthGate` usa `appAccessStream`; `SupabaseAuthGateway._onAuthStateChange` mantém acesso se Keystore tem sessão |
| Teste widget offline | ✅ CONFERE | `auth_gate_offline_test.dart` — sessionStream null + persisted → home |

## Veredito

Implementação aderente ao escopo ENI-84. Offline-first no campo preservado: Keystore + `appAccessStream` mantêm operador no app; sync adia com falha de rede; 401 real online com refresh falho ainda encerra sessão. Verificação manual no G86 permanece recomendada.
